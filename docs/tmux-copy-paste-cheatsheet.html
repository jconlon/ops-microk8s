Tmux Copy/Paste

Workaround for apps (e.g. dot-agent-deck) that capture the mouse and break normal terminal copy/paste.

One-Time Setup

Enable vi-style copy mode permanently by adding to ~/.tmux.conf:

# ~/.tmux.conf
set -g mode-keys vi
set -g focus-events on
bind P run "xclip -selection clipboard -o | tmux load-buffer - && tmux paste-buffer"
Already configured. These lines are already present in ~/.tmux.conf. No setup needed for new sessions.

Copy Text from a Pane

Step Action
1 Ctrl+b [ — enter copy mode (look for [123/456] counter in top-right)
2 Arrow keys — move cursor to start of text
3 Space — begin selection (text highlights as you move)
4 Arrow keys — extend selection to end of text
5 Enter — copy selection to tmux clipboard (exits copy mode)

Paste

Destination Command
Same tmux pane Ctrl+b ]
System clipboard (Ctrl+V) tmux show-buffer | xclip -selection clipboard
pbpaste alias. The shell alias pbpaste='xclip -selection clipboard -o' reads from the system clipboard — it does not read from the tmux buffer. Use Ctrl+b ] to paste within tmux, or pipe tmux show-buffer to promote a tmux buffer to the system clipboard first.

Paste from System Clipboard into a Pane

When the app (e.g. dot-agent-deck) captures the mouse, Shift+Ctrl+V is blocked. Use the tmux buffer as a relay instead.

Method Command
Keybinding (recommended) Ctrl+b P — loads system clipboard into tmux buffer and pastes immediately
Manual xclip -selection clipboard -o | tmux load-buffer -
then Ctrl+b ] to paste

The keybinding is defined in ~/.tmux.conf:

bind P run "xclip -selection clipboard -o | tmux load-buffer - && tmux paste-buffer"
Reload config without restarting. After editing ~/.tmux.conf, apply changes in a live session with Ctrl+b : then type source-file ~/.tmux.conf.

Exit Copy Mode Without Copying

Press q or Escape.
