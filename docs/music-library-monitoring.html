Music Library Monitoring

Ceph RGW object storage monitoring via Prometheus and Grafana

Overview

The music library (72 GiB, 3,209 objects) is stored in Ceph Object Storage (RGW) and monitored through the existing Prometheus/Grafana stack. Pool-level metrics are available via Prometheus; per-bucket breakdown requires mc du.

Commands

Per-bucket usage

mc du ceph/music-library/
mc du ceph/restic-backups/
mc du ceph/test-bucket/

# All buckets
for bucket in music-library restic-backups test-bucket; do
  echo "=== $bucket ===" && mc du ceph/$bucket/
done

Prometheus query

curl -s 'https://prometheus.verticon.com/api/v1/query?query=ceph_pool_bytes_used{pool="rook-ceph-rgw.rgw.buckets.data"}' \
  | jq -r '.data.result[0].value[1]'

Cluster health check

kubectl get cephcluster -n rook-ceph -o jsonpath='{.status.ceph.health}'
kubectl get pods -n rook-ceph -l app=rook-ceph-rgw

Limitations

The Ceph RGW Prometheus exporter does not expose per-bucket metrics — this is a Ceph limitation, not a monitoring stack issue. Workarounds:

  • mc du — run manually or via systemd timer for per-bucket stats
  • RGW admin API — radosgw-admin bucket stats --bucket=music-library (requires RGW admin credentials)
  • Custom exporter — not implemented; would query RGW admin API and push per-bucket metrics to Prometheus
