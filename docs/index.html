About this repository

This repository is the single source of truth for cluster infrastructure. Everything the cluster runs — storage, databases, message queues, CI/CD, monitoring, registries — is defined here and deployed via ArgoCD GitOps. Changing something means opening a PR here; ArgoCD reconciles the cluster to match.

The cluster is an 8-node MicroK8s HA setup running on a mix of desktop hardware and Dell R320 servers. It serves as the shared platform for all Verticon projects. See Cluster Reference for hardware, networking, and service details.

In scope

  • Cluster provisioning & node management
  • Core platform services: ArgoCD, Rook/Ceph, MetalLB, Prometheus/Grafana, Loki, cert-manager
  • Shared infrastructure: PostgreSQL operator, Kafka/Strimzi, Harbor, Argo Workflows/Events
  • Cluster-wide secrets, RBAC, and networking
  • Infrastructure issues raised by projects (secrets, storage classes, ingress, service accounts)

Out of scope

  • Individual machine OS config and tooling → dotfiles repo
  • Project-specific application logic → each project's own repo

Quick reference

Health checks

just test                    # full e2e chainsaw suite — nodes, ceph, gpu, pg, argocd, loki
just test-suite <suite>      # single suite: cluster · storage · gpu · postgresql · argocd · loki
just node-status             # uptime + reboot-required for all 8 nodes
just pods-unhealthy          # non-running pods cluster-wide

Logs

just loki-syslog-node <node> [since] [filter]   # syslog by node label (2026-05-12+)
just loki-journal-check <node> [filter] [since]  # journal: warning+, crashes, OOM, logind
just loki-node-events <node> [since]             # content search (pre-label-era data)
just loki-reboot-history [since]                 # all node boot times
just loki-docs                                   # open Loki troubleshooting guide

Storage

just ceph-status             # Ceph cluster health
just pg-status               # CloudNativePG cluster status

CI/CD

just argo-status             # Argo Workflows pods
just argo-events-status      # EventBus, EventSource, Sensor status
just test-build-e2e          # end-to-end kaniko build → Harbor → PR

ArgoCD

ops argocd login             # authenticate
ops argocd list-app          # all applications and sync status
argocd app sync <app> --force
